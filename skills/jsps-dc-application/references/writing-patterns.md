# 采用级写法模式库(DC 申請内容ファイル)

> 来源:一份**採用級・情報分野 DC2** 范本(脱敏,只取结构/文风模式,不含任何个人信息或具体研究内容)+ 一份 **KAKENHI 写作 craft 指南**(技巧层,已按 DC 改写)。
> 用法:起草各节时对照本库的模式与好/坏例;**这些是可迁移的范式,不是可照抄的句子**。

---

## 节1 ·【２】(1) 研究の概要及び研究の位置づけ(1p)

**推荐骨架(范本同款):**
1. `研究課題名:〇〇` 一行(下线/太字)。
2. **概要**(全角 ~500字):研究对象 → 従来手法のギャップ → 仮説(配 **Fig1** 概念图)→ ②③④ 阶段构成 → 一句目标句(如"从〔现状〕到〔愿景〕的转换")。
3. **当該分野の状況**:用**一次情报**裏付け"世界が注目"——具体会议名·年·keynote 题目、代表论文(著者 et al. 年);客观数据(论文数增长/产业规模/社会影响)。
4. **当該分野の課題**:`第一に…。第二に…。第三に…。` 列举 2-3 个未解 gap,每个点出"为什么现有手段做不到"。
5. **本研究計画の着想に至った経緯**:**第一人称**(「申請者は…に接し、…と認識するに至った」);**每条洞见显式映射** `→研究(1)/(2)/(3)`;显出**跨学科**来源(范本融了认知神经科学/网络科学·流行病学/发达心理学)。

**要点:** 概要要能独立读懂(它是全文压缩);课题(gap)与后面研究(1)(2)(3)一一对应,埋好伏线。

## 节2 ·【２】(2) 研究目的・内容等(2p)

**推荐骨架:**
1. **研究目的**段:复述目标 + 列 2-3 个"要件(1)(2)(3)"(把大目标拆成必要条件)。配 **Fig2** 概念图(若干子研究环绕中心对象)。
2. **研究方法・研究内容**:`研究(1):〔小标题〕(年次括注,如 2027-2028前半)` 逐个写,每个给**具体方法** + **明示可行性**(DC2 关键:"申請者はすでに…予備実験に着手しており、…が観察され始めている")。
3. **③ 研究の特色・独創的な点**:`第一の特色は…にある。` / 第二 / 第三,每条用「**従来研究の限界 → 本研究の着想 → 独創点(太字)**」对比;至少一条呼应内阁府「**総合知**」(跨学科融合)。
4. **④ 申請者が担当する部分**:仅当课题属研究室活动一部分时,说明你主体担当哪块(**不是**合作 PI 分工)。
5. **⑤ 海外**:有受入外的国内外机关共同/滞在计划则具体到人名·机构。

## 节3 ·【３】人権の保護及び法令等の遵守への対応(1p)

- 不涉及人体/个人信息/遗传/动物/安保贸易等 → 一句 **「該当しない。」**(范本即如此)。
- 涉及 → 具体写承认手续(伦理委员会等)与对策,勿空泛。
- 这是合规项**非加分项**,简洁即可,别占篇幅。

## 节4 ·【４】研究遂行力の自己分析(2p)

**(1) 研究に関する自身の強み —— 先摆业绩,再用散文佐证:**
- 业绩**分块**:`学術雑誌論文(査読有)` / `(査読中)` / `国際会議論文(査読有・現地発表)`(`○` 标发表者) / `受賞・研究助成等の採択歴` / `セミナー招待講演`。
- **筆頭著者太字+下线**、标 **IF / JCR分区 / CORE rank**;按时间或重要性编号 `[1][2]…`。
- 再以**带框小标题**的散文逐条**回指编号**:`研究における主体性・発想力・問題解決力` / `知識の幅・深さ・技量` / `コミュニケーション力とプレゼンテーション力` / `その他(アウトリーチ力)`。每段引 `[1][4]` 等佐证,**按申请课题的子任务分组**(借自 KAKENHI craft)。

**(2) 今後研究者として更なる発展のため必要と考えている要素:**
- `要素1:…` / `要素2:…`,写未来所需能力(如 学際的知識の体系化 / 国際的研究ネットワーク / 独立した研究推進力)——这里承载"目指す研究者像",落到将来性 ②。

## 文体・表記の原則(借自 KAKENHI craft,DC 同样适用)

審査セット跨区分、含分野外委员 → 既让专家信服、也让外行看懂难度。
- **体言止め・箇条書き**提升视认性;**太字**仅给核心キーワード与独创点(滥用=失效)。
- **数値・具体例** > 空泛形容("2桁以上の高速化""3年連続採択")。**主张必附根拠**。
- **能动态优先**;第一人称能动性(「申請者は…主体的に…」)。
- 专业术语加**简释**;关键的「問い」用**二层结构**(见下)。
- 引用统一 `[N]` + 文末文献;**概要凝缩全文**、「問い→目的→独自性」逻辑链不断。

### 「問い」二层结构(强技巧)
同一个核心问题,写两层:
- **专家层(技术难度)**:点明它为什么在技术上未解、难在哪。
- **分野外层(日常类比)**:一句生活化类比让外行也懂其难。
> 例(脱敏示意):"〔技术层:为何 X 在内部表征层面不可观测〕——好比〔类比:想在汤里看出盐何时开始结晶,但只能尝最后的味道〕。"

## 好 / 坏例对照

| 维度 | ❌ 坏 | ✅ 好 |
|---|---|---|
| 状况裏付け | "本分野は世界中で注目されている。" | "〇〇は 2024 年 △△(会议)keynote の主题となり、関連論文は 3 年で 2 倍に増えた [3]。" |
| 独创性 | "本研究は独創的である。" | "従来は出力の事後監視に依存していた[*]。本研究は**内部表現の機序解析**により、行動に現れる前に兆候を捉える点に独創性がある。" |
| 业绩 | "多数の論文を発表した。" | "**筆頭著者**として国際会議論文 `[6]`(IEEE VTC, CORE B)を発表、現地登壇した。" |
| 主体性 | "指導教員と研究を進めた。" | "課題設定から実装・評価まで**主体的に**推進し、筆頭著者として `[1]` を公表した。" |
| 可行性(DC2) | "今後実験を行う予定である。" | "すでに予備実験に着手し、**層ごとの表現変化に差異が観察され始めている**。" |

## 反面红线(勿犯)
- 把**他人**采用申请书的句子/构想照抄 → 抄袭风险。本库只学**模式**。
- 堆业绩不回指课题、不显主体性 → ② 将来性立不住。
- 「独創的」「重要」只下结论不给根拠/对比 → 审查最忌空话。
- 用 KAKENHI 的"研究分担者/全体5頁/種目"框架套 DC → 结构性错误(见 SKILL.md Key Rules)。
