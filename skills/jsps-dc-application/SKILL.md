# JSPS 特別研究員-DC 申請内容ファイル 執筆

为 DC1/DC2 申请人撰写**采用级**的「申請内容ファイル」。这是**对"人"的评选**(研究者の資質・将来性),不是对项目的评选——全程以此为轴。

## 这份申请书的本质(为什么这样写)

審査の観点只有两条(权威原文见 `references/reference.md` §5):
1. **①** 研究課題設定に至る背景が示され、**着想が優れている**;**研究方法にオリジナリティ**があり、**今後の展望**が示されている。
2. **②** **学術の将来を担う優れた研究者**となることが十分期待できる。

→ 节1+节2 把"背景→着想の優越性→方法の独創性→展望"讲成一条逻辑链来满足 ①;节4 用业绩把"我是/会成为优秀研究者"落到实处来满足 ②。**第一人称、能动性、独创性、将来性是主轴**。

## 触发与边界

- ✅ 本 skill:特別研究員 **DC1 / DC2** 的申請内容ファイル(起草/润色/单节重写/日英翻译)。
- ❌ KAKENHI 项目经费(基盤A/B/C・若手・挑戦的)→ 用 `grant-proposal`。理由:DC 是 fellowship、用 DC 専用様式、审查对"人"不对"项目"、单独申请无研究分担者。
- ❌ 特別研究員 PD/RPD → 本 skill 只覆盖 DC(PD 加 ③④ 审查条款、受入机关限制不同)。

## 开工前先读参考库(progressive disclosure)

正文只给流程;**权威细节在参考文件,按需读**:
- `references/reference.md` —— 様式精确结构(§8)、格式硬约束(§3)、A/B区分(§4)、**審査の観点(§5)**、DC1/DC2 差异(§6)、采用级写法模式(§10)。**起草前必读 §8 与 §5。**
- `references/writing-patterns.md` —— 采用级写法模式 + 借自 KAKENHI craft 的文体技巧 + 好/坏例对照。**起草前必读。**
- `references/dc1-vs-dc2.md` —— DC1 vs DC2 侧重、日/英措辞与敬体规范。**按区分与语言读对应段。**
- `references/official/` —— 作成要領、書面審査の手引(令和9/2027)原件,**遇到格式/审查疑问以此为准**。
- `assets/latex-template/` —— 科研費LaTeX DC multi 版骨架(各节已含官方指示文 + 页数控制宏)。

## 申請内容ファイルの結構(从【２】开始;【１】是 Web 输入的研究課題名)

| 节 | 页上限 | 写什么 |
|---|---|---|
| 【２】(1) 研究の概要及び研究の位置づけ | **1p** | 概要(全角 **500字**程度)+ 当該分野の状況・課題 + **着想に至った経緯** |
| 【２】(2) 研究目的・内容等 | **2p** | ①目的・方法・内容 ②年次計画(应募区分 A/B)+ 不顺时的对应 ③**特色・独創的な点** ④担当部分 ⑤海外 |
| 【３】人権の保護及び法令等の遵守への対応 | **1p** | 涉及伦理/个人信息/安保贸易时记対応;**不涉及写「該当しない。」** |
| 【４】研究遂行力の自己分析 | **2p** | ①研究に関する自身の強み(用业绩佐证)②今後必要と考えている要素 |

页数由様式固定(1/2/1/2,共 ~6p),**超出即被模板宏截断**——精简而非硬塞。

> 🔴 **硬性要求:各栏必须写满至接近页数上限。** 节1≈1p、节2≈2p、节4≈2p 须充分占用版面;**留白半页以上 = 准备不足的负面信号,必须加筆**(深化背景/方法细节/期待成果/业绩与自己分析)。唯一例外:节3「人権」若「該当しない」可较短,但仍应补数据来源・研究公正/再現性への対応等以利用版面。起草后**必须逐节检查填充度**,不足则扩写到接近満杯。

## ワークフロー(每个 🚦 停下等用户确认,勿自动推进)

### Step 0 · 情报收集
解析参数:区分(DC1/DC2)、语言(日/英)、研究方向。读项目内 CV/业绩/idea 文档补全;缺的列 `[要確認]`。确认:**奨励費 A/B区分**、受入研究機関+指導教員、海外共同有无。业绩不足时 researchmap/dblp/Google Scholar 检索补全(注意**著者名揺れ**:罗马字↔日语,中文名尤甚)。

### Step 1 · 【２】(1) 研究の位置づけ 🚦
搭骨架:课题名 → 概要(~500字:对象→従来手法のギャップ→仮説〔配 Fig1〕→构成→目标句)→ **当該分野の状況**(一次情报:会议名·年·keynote;客观数据)→ **当該分野の課題**(第一に/第二に/第三に 列 gap)→ **着想に至った経緯**(第一人称,每条洞见映射到 研究(1)(2)(3),显跨学科来源)。对照 `writing-patterns.md`。**停**,确认定位与「问い/gap」。

### Step 2 · 【２】(2) 研究計画设计 🚦最关键
设:研究目的(复述目标+要件)→ 研究(1)(2)(3) 各带**年次**与具体方法 → **③特色・独創的な点**(用「従来の限界→着想→独創点(太字)」逐条对比,呼应「**総合知**」)→ ④担当部分(若属研究室活动)→ ⑤海外。把经费映射到 **A区分(全员)+ 可选 B区分相違点**。**停**确认结构。

### Step 3 · 全文起草(LaTeX)+ 图件
按 `assets/latex-template/` 各节填稿,套 `writing-patterns.md` 的采用级模式。**图件为默认必备**(见下「图件」):**checkpoint 只问画哪几张,不问要不要画**。【３】默认「該当しない。」(据情可改)。【４】业绩按 査読有/査読中/国際会議(現地発表○)/受賞/招待講演 分块,**筆頭著者太字+下线、标 IF·CORE rank**,再以带框小标题散文回指业绩编号 `[N]`。

### Step 4 · 自检(对照标尺)
逐条核 `references/reference.md` §5 審査の観点 + §3 格式硬约束(下「自检清单」)。若 `mcp__codex__codex` 可用,可选:让它扮 DC 审查委员按 ①② 打分给意见(opt-in,无 API 则跳过)。

### Step 5 · 编译产出 🚦
`platex dc.tex && dvipdfmx dc.dvi`(或 uplatex,见模板注释);**超页则提示精简**;输出 PDF + .tex。过终检清单后交付。

## 图件(DC 几乎都是概念図/構成図,不是数据图)

审查委员看**灰度打印件**且需**日文标签** → **可编辑矢量 > AI 位图**。按类型用本地 skill:

| 图类型 | 用 | 何时 |
|---|---|---|
| **概念図·構成図·流程**(Fig1/Fig2,主力) | **`figure-spec`**(可编辑 SVG·CJK 标签·灰度安全·无需 API) | 默认。Fig1 随 Step 2 出以锚定逻辑,Fig2 在 Step 3 |
| 年次計画·甘特·简单流程 | `mermaid-diagram` | 需要时 |
| 予備実験データ图(若【４】放预备结果) | `nature-figure` / `top-journal-scientific-figures` | 有真实数据时 |
| 更"美"的概念插画 | `scientific-schematics` / `paper-illustration`(AI 位图) | **仅 opt-in**:位图不可微调、AI 渲日文易错、需 API key、需人工核标签 |

`figure-spec` 出的 SVG 转 PDF/EPS 内嵌 LaTeX。**默认 `figure-spec`,AI 位图仅用户点名时用。**

## 文体・表記の原則(借自 KAKENHI craft,DC 同样适用)

審査セット跨区分、有分野外委员 → 既要专家信服、也要外行看懂。
- **体言止め・箇条書き**提升视认性;**太字**强调核心キーワード与独创点。
- **数値・具体例**(「2桁以上の高速化」「3年連続」)优于空泛主张;主张必附根拠。
- **能动态 > 受动态**;第一人称能动性贯穿(「申請者は…を主体的に…」)。
- 专业术语加**简洁说明**;「問い」用**二层结构**(专家向技术难度 + 分野外向类比)。
- 引用统一 `[N]`,文末附文献;**概要凝缩本文**、「問い→目的→独自性」逻辑连贯。

## Key Rules(守门)

- **AI 辅助写"本人"申请书 = 正当**。書面審査の手引「申請書を生成 AI に入力するな」约束的是**审查委员**处理**他人**机密申请书,与申请人起草自己的材料是两回事——可放心使用,但不得把**他人**的申请书/机密材料喂进来。
- **不造假**:只用用户提供或可核实(researchmap/dblp/scholar)的真实业绩/引用;不确定标 `[要確認]`,**绝不编造**论文/获奖/数据/引用。
- **页数按 DC 节别 1/2/1/2,不是 KAKENHI 的"全体5頁"**;超出提示精简,不偷偷截断。
- **DC 是单独 fellowship**——**不引入「研究分担者」概念**;【２】(2)④的"担当部分"指课题属研究室活动时申请者担当哪块,非合作 PI 分工。
- 種目是 **DC1/DC2 + 奨励費 A/B区分**,不是基盤/若手/挑戦的。
- **研究課題名提交后一字不可改**(与奖励费课题名绑定)——起草时提醒用户慎定。
- **灰度可辨**:图表勿靠颜色区分(审查看单色打印)。
- 様式**不可加工/变更**;提交前删除様式中的斜体说明文与 A/B区分イメージ图;字号 **≥10pt**;申請内容ファイル PDF **≤3MB**。
- DC1 重**构想独创性+潜力**(业绩通常薄);DC2 重**既有成果+予备实验可行性+主体性**。

## 自检清单(交付前逐条过)

- [ ] 4 节齐全;各节**写满且未超页**(节1≈1p・节2≈2p・节4≈2p 接近満杯、无半页留白;节3可较短)
- [ ] 概要 ~500字且凝缩全文;Fig1/Fig2 在位且**灰度可辨、日文标签清晰**
- [ ] ① 背景—着想—方法独創—展望 成链;③ 用「従来→着想→独創点(太字)」对比、呼应総合知
- [ ] ④ 自己分析以业绩(真实、标 IF/CORE、筆頭著者标记)佐证强み;② 要素面向将来
- [ ] 引用 `[N]` 与文末文献一一对应;**无编造**;`[要確認]` 已清理或交用户确认
- [ ] 字号≥10pt、删斜体说明文、未改様式、PDF≤3MB、研究課題名已与用户确认
- [ ] 语言统一(日 or 英);若日语用である調学术敬体(见 `dc1-vs-dc2.md`)

## LaTeX 产出

骨架在 `assets/latex-template/`(科研費LaTeX,出处见 `CREDIT.md`)。按节填:
- `dc_01_background.tex`=【２】(1) · `dc_02_purpose_plan.tex`=【２】(2) · `dc_03_rights.tex`=【３】 · `dc_04_abilities.tex`=【４】
- 主文件 `dc.tex` 已 `\input` 各节;业绩用其 `\paper{title}{authors}{journal}{vol}{pages}{year}` 宏 + `\label/\ref` 交叉引用。
- 提交前删除模板里的示范文字与斜体 `\JSPSInstructions`/`\SelfReviewInstructions` 留意事项。
- 编译产物拷到用户项目目录(勿污染 skill 包);英文申请同一套模板,正文改英文即可。

## 注意事項

- 每次起草前**核对 `references/official/` 是否仍是当年度版本**(様式逐年微调);若用户申请的是更晚批次,提示去 jsps.go.jp / 科研費LaTeX 取新様式。
- 这是辅助起草:**研究内容、业绩真伪、最终定稿由申请人负责**;skill 不替用户决定科学主张。
